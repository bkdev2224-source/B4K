# B4K

Next.js travel app for Korea exploration.

## Development

```bash
npm install
npm run dev
```

## Commit Message Convention

Format: `[type] description`

- **Description**: lowercase, imperative mood (e.g. "add X" not "added X")
- **Multiple changes**: use semicolons, e.g. `[feat] add X; improve Y`

### Types

| Type | Use for |
|------|---------|
| feat | New feature or capability |
| fix | Bug fix |
| perf | Performance improvement |
| a11y | Accessibility |
| security | Security-related changes |
| refactor | Code restructure, no behavior change |
| style | Formatting, layout, visual only |
| docs | Documentation |
| test | Tests |
| chore | Lint, deps, config, tooling |

### Examples

```
[feat] add route panel on maps
[fix] resolve nested button hydration error
[perf] add SWR for client-side data fetching
[a11y] improve overlays, labels, and decorative SVGs
[security] add API rate limiting via Upstash Redis
[docs] add commit message convention to README
[chore] move playwright to devDependencies
```
